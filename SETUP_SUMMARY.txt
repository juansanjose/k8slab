HOW IT WORKS
------------

1. Vast.ai container starts
2. Installs Tailscale
3. Joins your Tailnet
4. Installs k3s agent
5. Configures for container environment:
   - Native snapshotter (no overlayfs)
   - cgroupfs driver
   - Fake /dev/kmsg
6. Joins cluster as real node
7. Can reach all cluster services

QUICK START
-----------

1. Prepare laptop:
   sudo bash scripts/setup-k3s-server-for-workers.sh

2. Get credentials:
   export VASTAI_KEY="your-key"
   export TS_AUTHKEY="tskey-auth-xxx"
   export K3S_URL="https://100.87.186.22:6443"
   export K3S_TOKEN="K10xxx"

3. Create worker:
   bash scripts/create-vastai-worker.sh

4. Verify:
   kubectl get nodes -w

5. Run GPU job:
   kubectl apply -f mlops-lab/examples/gpu-training-real-node.yaml
   kubectl logs -f job/gpu-training-real-node

IMPORTANT NOTES
---------------

- Vast.ai has a CDI device injection bug that may prevent containers
  from starting. If you see "unresolvable CDI devices", try a 
  different GPU model or contact Vast.ai support.

- The worker setup handles container limitations:
  * No systemd -> runs processes directly
  * No overlayfs -> uses native snapshotter
  * No /dev/kmsg -> creates fake device
  * Limited cgroups -> uses cgroupfs driver

- Worker nodes are ephemeral - if the Vast.ai container stops,
  the node disappears from the cluster.
