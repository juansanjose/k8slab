LLM Training on Vast.ai - Complete Setup
=========================================

ARCHITECTURE
------------

Your Laptop (panzamachine)
    |
    | kubectl apply -f quick-test.yaml
    v
k3s Cluster
    |
    | vastai-kubelet sees pod -> calls Vast.ai API
    v
Vast.ai Cloud API (Internet)
    |
    | Creates GPU container in their datacenter
    v
GPU Container (RTX 4090 / A100)
    |
    |-- Downloads model from HuggingFace
    |-- Downloads dataset
    |-- Fine-tunes with LoRA
    |-- Saves results
    v
Container stops -> Instance destroyed

QUICK START
-----------

1. Quick Test (5 min, ~$0.03):
   cd mlops-lab/examples
   kubectl apply -f quick-test.yaml
   kubectl logs -f job/quick-llm-test

2. Full Training (30 min, ~$0.15):
   kubectl apply -f tinyllama-training.yaml
   kubectl logs -f job/llm-training-tinyllama

3. Custom Training:
   Edit mlops-lab/training-scripts/train_llm.py
   docker build -t localhost:5000/my-trainer mlops-lab/training-scripts/
   docker push localhost:5000/my-trainer
   Update job YAML with your image

HOW IT WORKS
------------

1. You submit a Kubernetes Job with nodeName: virtual-vastai
2. The vastai-kubelet (in your cluster) sees the pod
3. It searches Vast.ai for cheapest GPU matching your criteria
4. Calls Vast.ai API to create instance
5. Container downloads model + data from HuggingFace
6. Training runs inside Vast.ai container
7. Container stops -> vastai-kubelet destroys instance

IMPORTANT: Containers are ISOLATED
- They have internet access (can reach HuggingFace)
- They CANNOT reach your local MLflow/MinIO
- To save models, use HuggingFace Hub or cloud storage

GETTING MODELS OUT
------------------

Option 1: HuggingFace Hub (Easiest)
   Create secret: kubectl create secret generic hf-token --from-literal=token=hf_xxx
   Uncomment HF_TOKEN, PUSH_TO_HUB, HF_REPO_ID in job YAML
   Model uploads automatically after training

Option 2: Custom Upload Script
   Add AWS/GCP/Azure CLI to container
   Upload to cloud storage in training script

Option 3: Manual Download
   Keep instance running
   SSH via Vast.ai web UI
   scp files from instance

MONITORING
----------

  Watch controller:  kubectl logs -n vastai-system deployment/vastai-kubelet -f
  Watch training:    kubectl logs -f job/quick-llm-test
  Check status:      kubectl get job quick-llm-test
  Check instances:   curl -H "Authorization: Bearer $VASTAI_KEY" \
                     https://console.vast.ai/api/v0/instances/
